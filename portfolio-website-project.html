Portfolio Website

Figma · Illustrator · Photoshop · HTML · CSS · JavaScript · Pencil & Paper

Project Overview

A portfolio should also reflect good UX design, so I decided to turn my portfolio into its own UX/UI project. Not only as a way to make my portfolio a great user experience, but also as a way to reflect on my skills and see where I can continue to improve.

Goals

  • Encourage users to want to work with me.
  • Create a meaningful user experience for those viewing my portfolio
  • Learn how to identify myself as a designer to the world.

Challenges

  • Presenting projects within user time constraints.
  • Standing out from other UX/UI portfolios.
  • Increasing findability of projects relevant to user needs.

Solutions

  • Present a clear sequence for users to follow.
  • Develop personal branding and create a unique visual experience that stands out in user minds.
  • Offer shortcuts to the information that users need.

Process

Research

Who is my user?

I identified 3 main users as my target audience. These are recruiters, designers, and potential clients who are interested in hiring a designer.

User Interviews

I designed a detailed survey about people's experience viewing portfolio websites. Using linkedin I reached out to potential users and asked a series of questions to better empathize with them and understand what their pain points were throughout the process.

After receiving over 50 responses, some of the main findings from the user interviews included:

  • Users generally find the navigation process of many portfolios to be tedious.
  • Some users will go through every project in detail, but most quickly skim through the first project they can find.
  • Most users are viewing portfolios at work on their desktop or laptop computers.
  • Users want more personalized guidance to find information relevant to their needs.

User Personas

I then created user personas for each of my 3 users based on data gathered from the interviews/surveys.

Competitive Analysis

I did extensive competitive analysis by gathering a long list of portfolios from designers of every level and approached them from the perspective of my user personas. Some of the main things I looked for included:

  • How did they handle the challenges I identified?
  • What do I dislike about their portfolio?
  • Which strengths are they focusing on showing?

New Challenge

How would I display and host this custom design for my portfolio? Websites like Behance, Dribble, and Coroflot allow you to showcase your design work, but you are forced to use their layout with all its limitations.

Websites like Squarespace can be good, but you are forced to choose from their templates. The majority of portfolios I analyzed used very similar templates from these websites and they all start to feel the same after awhile. Even worse, many of them simply don't have good UX and make it difficult to navigate. If you are looking through hundreds of candidates for a position, these don't stand out at all.

Ultimately I was not going to settle for the easy way out and decided I need to code the website myself from scratch. This was the only way to truly give the website my own personality and give a meaningful user experience to all who visit.

Affinity Mapping

Combining the results from user interviews, surveys, and competitive analysis. I was able to refine the results and analyze them using an affinity map.

I discovered that my users generally need help in the following areas:

  • Navigation: How do they find the information they are looking for?
  • Time: How can I help them move through the website more quickly?
  • Interest: How do I let them see more of my strengths?

Ideation

Sketching

In order to begin developing concepts from my research, I decided to sketch quick ideas individually before merging them together. Below you can see sketches showing different layout ideas for the website.

Initially my thinking was to put as much as possible on one home page. This way users didn't have to waste time clicking through multiple pages to find projects that were relevant to their needs.

Ultimately I decided to settle on 3 main pages when it came to my information architecture. Home, About, and Project:

  • My home page needed to present links to all of my projects as quickly as possible, but I need to be careful as it easily feels crowded which leads to confusion on next steps.
  • It felt best to make sure my users had access to resume/contact links from every page, rather than only having it available on my about page.
  • The individual project pages needed to present the tools and skills used at the start so users immediately know if the project fits their needs. Removing any unnecessary scrolling to search for that info.

Wireframes

I then designed low fidelity wireframes to begin merging all of my ideas and research insights, noting the journey or user flow my users would take through each design.

There were a few notable issues with my first set of wireframes:

  • Home Page: Project cards feel crowded, especially when you start displaying info on top of the image. My intro statement is also pushed off to the side in an awkward position.
  • About Page: Lacks direction, what should users think and feel after visiting this page?
  • Project Page: What motivation do users have when they reach the end of the project? I want them to view more projects so how do I encourage that?

Which gave me a new set of solutions to try and implement in the next set of wireframes:

  • Home Page: Move the cards down and give them more space to display info.
  • About Page: Add a call to action that leads them back to my projects or ways to contact me.
  • Project Page: Add a side navigation that allows them to move through the project more quickly giving them more time to check out other projects.

High Fidelity Wireframes

Now we can really start to see the ideas and solutions come together. I designed a more detailed set of wireframes based on findings from the last set.

This led to a new set of challenges, but with a much clearer direction for next design steps:

  • Home Page: Display projects in a column rather than a row across, this way I can display info as cleanly as possible and not cause any confusion on what to click.
  • About Page: Redesign the buttons and adjust the layout of the page. Add contact links to a footer at the bottom rather than a button.
  • Project Page: Add a next project section to the bottom of the page that invites the user to keep viewing more projects seamlessly with no need to go backwards.

Design

Branding

One of my main goals in taking on this portfolio project was to learn how to identify myself as a designer. Personal branding is a great way of doing this while also solving one of my biggest challenges, which is standing out from other design portfolios. I started this process by designing my own logo that you can see throughout my website.

Style Guide

I then created a style guide to follow and make sure my branding is consistent as I design the final iterations of my portfolio design.

Portfolio Design

After I established my personal branding, it was time to put together the final design that addresses all of the challenges and pain points for end users that my research brought to light.

One of the biggest challenges for my users was how much time they have to view portfolios and research showed the first thing most do is look for a list of projects. So this was an area that I spent a lot of time focusing on. They should be able to understand what the project accomplished and what skills/tools were used from this page. Giving them just enough info to quickly decide which projects they want to view, allowing for more efficient clicks.

An extra advantage of giving my project list more space, is the ability to add extra buttons if I happen to code the project myself and want to show it or to simply have a direct link to the finished product if my users just want to see the final result.

User Flow

Another issue I needed to solve was the journey that my users take through the website. This design suggests your next step with each page you visit, but allows you the freedom to go in any order you like. Whether you'd like to see my projects first, learn more about me before you see the work, or go straight to my resume.

Responsive Design

My user interviews taught me that the majority of my users are viewing portfolios at work on their desktop or laptop. However enough of them will also check from their mobile devices while commuting to work or during a meeting. This meant that it was extremly important for me to design and code the website to be responsive across all devices so I do not lose any potential mobile users.

Evaluation

HTML & CSS Coding

The part of this project that required the most work was everything that most won't see. I built the website from scratch in HTML and had to make sure I could style everything exactly how I planned in my design phase through CSS. This was a back and forth process of testing the design in code and adjusting as needed.

Whenever I came across a situation where I did not know how to code something I would stop and go learn it. Often I would learn more about my design through this process and that allowed me to correct flaws in my initial designs. One of the best examples of this was making the website responsive across all devices and how I needed to redesign many elements to fit that.

Most importantly, one of the main reasons I wanted to design my own portfolio was to reflect on my skills and see where I can continue to improve. As a UX/UI designer I find it invaluable to have improved my coding skills through this project so that I can better design meaningful experiences and collaborate even better with developers.

Usability Testing

Just like in the beginning stages, with a now complete custom portfolio, I am reaching out to potential users and performing usability tests on the final product. This is largely to give me insights on how my design choices are performing and what can continue to be improved.

Initial feedback has been very positive so far, however the testing process has only just started. There will be great opportunities to incoroporate new ideas with more extensive usability testing and iteration. So please look forward to seeing them with a detailed testing report in the future!

Moving Forward

This projects focus was more on UX and Coding. The Coding especially took up a lot of time that normally would have gone to UI development. So as I continue to test the results one thing I definitely want to focus on improving is the UI elements of the website. With my personal branding set up I am excited to continue exploring and developing options.

Another area I'm already planning to improve is my side navigation on project pages. This could be more detailed and allow you to jump to specific areas within each section, such as going straight to the wireframes part of my ideation section rather than only the beginning of the ideation section.

Final Thoughts

The process behind this project was so much fun to go through because of how much I learned. Not only did I become very comfortable coding HTML/CSS, I even started learning JavaScript and managed to incorporate a bit of it into my about page. This experience has definitely motivated me to continue learning more and I will surely be incoporating everything I learn into future versions of this website and other projects.
